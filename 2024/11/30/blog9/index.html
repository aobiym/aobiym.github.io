Zookeeper的安装

Created2024-11-30|Updated2024-11-30|教程
|Post Views:

安装 Java JDK

> 己经安装好了，只要找到目录在及修改档案，三台都要改！

# which java

>>/usr/bin/java

# cd /usr/bin

# ls -la java*

>> java -> /etc/alternatives/java

# cd /etc/alternatives/

# ls -la java*

>> java -> /usr/lib/jvm/java-8-openjdk-amd64/jre/bin/java

# cd /usr/lib/jvm/java-8-openjdk-amd64/jre

以上才是真正的 java home 所在！

> edit env. path

# vi /etc/profile

-—

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64/jre

export CLASSPATH=$JAVA_HOME/lib/

export PATH=$PATH:$JAVA_HOME/bin

-—

>> confirm

# source /etc/profile

# java -version

安装 zookeeper

>> 复制档案

$ docker cp apache-zookeeper-3.8.0-bin.tar.gz hmaster:/home/ubuntu

> into docker container

$ docker exec -it hmaster bash

# cd /home/ubuntu/

# mkdir -p /usr/zookeeper

# tar -zxvf apache-zookeeper-3.8.0-bin.tar.gz -C /usr/zookeeper/ –strip-components 1

>> 配置文件 conf/zoo.cfg

# cd /usr/zookeeper/conf

# cp zoo_sample.cfg zoo.cfg

# vi zoo.cfg

-—

dataDir=/usr/zookeeper/zkdata

dataLogDir=/usr/zookeeper/zkdatalog

server.1=hmaster:2888:3888

server.2=hslave1:2888:3888

server.3=hslave2:2888:3888

-—

>> 在 zookeeper 的目录中，创建 zkdata 和 zkdatalog 两个文件夹

# cd usr/zookeeper/

# mkdir zkdata

# mkdir zkdatalog

>> 进入 zkdata 文件夹，创建文件 myid

# cd zkdata

# vi myid

-—

1

-—

>> 远程复制分发安装文件, 改 hslavex1 and hslavex2 为自己的 hslavexx

# scp -r /usr/zookeeper root@hslave1:/usr/

# scp -r /usr/zookeeper root@hslave2:/usr/

> 设置 myid。

在我们配置的 dataDir 指定的目录下面，创建一个 myid 文件，里面内容为一个数字，用来标识当前主机，conf/zoo.cfg 文件中配置的 server.X 中 X 为什么数字，则 myid 文件中就

myid 文件中就输入这个数字。slavex1 中为 2;slavex2 中为 3。

#docker exec -it hslave1 bash

# cd usr/zookeeper/

# cd zkdata

# vi myid

（将里面的数字改为2）

然后进入hslave2进行修改

配置环境变量并启动 ZooKeeper（三台都要）

# vi /etc/profile

-—

#set zookeeper environment

export ZOOKEEPER_HOME=/usr/zookeeper

export PATH=$PATH:$ZOOKEEPER_HOME/bin

# source /etc/profile

> 启动 ZooKeeper 集群 (三台都要 run! )

# cd /usr/zookeeper（要在usr/zookeeper/下，即zookeeper安装目录下才能启动

# bin/zkServer.sh start

# bin/zkServer.sh status（三台都启动完才能查看）
